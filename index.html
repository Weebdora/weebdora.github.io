Basic html concepts

Block element

Block-level elements form a visible block on a page. A block-level element appears on a new line following the content that precedes it. Any content that follows a block-level element also appears on a new line

Inline element

Inline elements are contained within block-level elements and will not cause a new line to appear in the document

Attributes

Attributes contain extra information about the element that won't appear in the content. To declare an attribute, insert it's name followed by an equal name, and it's value inside quotation marks.

Empty Elements

Not all elements follow the pattern of an opening tag, content, and a closing tag. Some elements consist of a single tag, which is typically used to insert/embed something in the document like img.

Nested Elements

Elements can be placed within other elements. This is called nesting. The tags have to open and close in a way that they are inside or outside one another

Boolean attributes

Attributes written without values or only one value which is generally the same as the attribute name

Whitespace

No matter how much space or line breaks you insert in html, it only considers it equal to a single space. This helps in improving the readability of the code. It is a common practice to have each nested element indented by two spaces more than the one it is sitting inside. In case, if you do want the whitespace to appear on the page, you can use the preformatted text element <pre></pre> for retaining whitespace

Special characters

Because browsers usually interpret special characters as code, there's a special codes to write them in html. These special characters are greater than(gt), less than(lt), single quotes(apos), double quotes(quot), and ampersand. To write them, you have to put their abbreviation, provided in round brackets next to them, between an ampersand sign and a semicolon. So they will appear like this < > ' " &

Comments

Comments aren't considered as code and are ignored by browsers. They are usually made to define what the code does or deliever a message to other developers. Syntax: <!--Anything between is a comment and will not be shown to the client/user.-->

Necessary html elements info

  • Doctype declaration. The first line of code required in every HTML document.The DOCTYPE declaration is an instruction to the web browser about what version of HTML the page is written in. This ensures that the web page is parsed the same way by different web browsers.

  • Html element. It represents the root level element and wraps all the content on the page. All other elements must be descendants of this element. It is a common practice to add 'lang' attribute to set the language of your page, for example, lang="en-US". It will be in your best interests to do this as it will help the search engine in indexing your HTML document.

  • Head element. This includes keywords and a page description that would appear in search results, CSS to style content, JavaScript to add interactivity, character set declarations, and other metadata. The content in the head is not visible to the users.

  • Body element. It contains all the content that displays on the page, including text, images, videos, games, playable audio tracks, or whatever else.

  • Meta element. This contains metadata which describes what type of data is displayed on the web page.

  • Title element.This sets the title of the page and is also used to describe the page when it is bookmarked

What common attributes, elements and other common things are included inside discussed elements

Meta

  1. charset="utf-8". Specifies document's character encoding. The set of characters the document is permitted to use. utf-8 is a universal character set that includes pretty much any character from any human language. This means that your web page will be able to handle displaying any language

  2. name, content. These attributes are included in many meta elements. Name specifies the type of meta element, what type of information it contains, and content specifies the actual meta content.
